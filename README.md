**Парсер сайта hh.ru по вакансиям + статистика + графики + кэширование**

**ЗАДАЧА:** Нужно организовать поиск вакансий на hh.ru с помощью Python и добавить
- Реализацию асинхронных запросов 
- Реализацию кэширования: при повторном поиске не тратить время на запрос
- Реализацию анализа вакансий - Статистика
- Реализацию графической статистики найденных вакансий

**ИНСТРУМЕНТЫ**
- Python - Язык программирования, на котором написан код
- aiohttp - Асинхронные запросы к сайту hh.ru (быстро качает данные)
- beautifulsoup4 - Парсинг HTML — вытаскивает из страниц названия, зарплаты, компании
- tqdm - Отображение прогресс-баров при парсинге
- matplotlib - Построение графиков
- SQLite - Для хранения скачанных вакансий
- pyproject.toml - Для настройки проекта (Чтобы установить из dependencies нужно выполнить: pip install -e .)

**ТЕХНОЛОГИИ**
- asyncio — для асинхронного кода
- sqlite3 — работа с БД
- datetime — работа с датой и временем
- re — регулярные выражения (для парсинга чисел)
- typing — подсказки типов (List, Dict, Optional)
- argparse — парсинг аргументов командной строки
- collections — счетчики для статистики (Counter, defaultdict)

**НАЗНАЧЕНИЕ МОДУЛЕЙ В ПРОЕКТЕ**
1. МОДУЛЬ bd.py (База данных)
Назначение:
Управление хранением вакансий в SQLite базе данных. Обеспечивает сохранение, загрузку и кеширование результатов парсинга.

2. МОДУЛЬ hh_pars.py (Парсер hh.ru)
Назначение:
Асинхронный парсинг вакансий с сайта hh.ru. Извлекает информацию о вакансиях, компаниях, зарплатах и т.д.

3. МОДУЛЬ stats.py (Статистика)
Назначение:
Расчет и хранение статистических данных по вакансиям.

rm -rf hhru_parser/__pycache__

**КОМАНДЫ ПРИЛОЖЕНИЯ**
# 1. Посмотреть справку приложения
hhru-parser --help

# 2. Команда парсинга вакансий на hh.ru (ПРИМЕР):
hhru-parser "ML Engineer AND NLP" --pages 2 --stats --grafik

**КЛЮЧИ КОМАНДЫ**
--pages

Что делает: Указывает, сколько страниц результатов нужно спарсить
Значение: Число (по умолчанию = 1)
Как работает:

hh.ru показывает по 20 вакансий на странице
Если указать --pages 3, то спарсит 3 страницы = до 60 вакансий

--stats

Что делает: Показывает статистику по найденным вакансиям
Значение: Флаг (просто указывается, без числа)
Что показывает:

Топ-5 компаний (где больше всего вакансий)
Распределение зарплат по диапазонам (0-50, 50-100, 100-150, 150-200, 200+)
Форматы работы (удаленно, офис, гибрид)
Топ-3 города
Среднее количество откликов на вакансию

--grafik

Что делает: Строит графики на основе данных
Значение: Флаг (просто указывается, без числа)
Какие графики строит:

График 1: Топ-10 компаний (горизонтальные столбцы)
График 2: Распределение откликов (гистограмма)
График 3: Форматы работы (круговая диаграмма)
График 4: Зарплаты по компаниям

--per-page

Что делает: Количество вакансий на странице (топ-N)
Значение: Число (по умолчанию = 5)
Как работает: Показывает только первые N вакансий с каждой страницы

--no-cache

Что делает: Не использовать кешированные данные
Значение: Флаг (просто указывается, без числа)
Как работает: Принудительно парсит новые данные с сайта, игнорируя сохраненные в БД
